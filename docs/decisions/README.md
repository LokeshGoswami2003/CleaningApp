# Architecture Decisions

This folder stores Architecture Decision Records (ADRs).

Add an ADR when a decision:

- Changes backend architecture.
- Changes database design.
- Changes deployment/runtime strategy.
- Changes API contract.
- Adds or removes MVP scope.

ADR format:

```text
# ADR-0000: Decision Title

Date: YYYY-MM-DD
Status: Proposed | Accepted | Superseded

## Context
## Decision
## Consequences
## References
```
